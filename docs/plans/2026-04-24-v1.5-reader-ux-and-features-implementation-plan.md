(3) Frontend `npm run build` + `npm run type-check` succeed; unit tests for new composables/stores/components pass. (4) Playwright E2E scripts for the 5 primary journeys (Custom theme, AI tag seeding, cross-section annotation, tag CRUD, Ask AI) pass. (6) Manual spot-check on the Gutenberg Porter book shows: image placeholders render in Summary, single-bar blockquotes in Original, Night preset saves, highlights visible inline, tag chips work, book overview page renders, next/prev updates URL.

## Code Study Notes

Key patterns inherited from the Phase 2 code study:

**Alembic:** `env.py` runs async + `render_as_batch=True`. Data migrations use `conn.execute(sa.text(SQL), {bind_params})` with structlog logging. v1.4b pattern: pre-backup via `shutil.copy2` → iterate with P-3 guards → log aggregate counts. All migrations idempotent.

**Tests:** `conftest.py` provides `engine` (async, FTS5 triggers) + `db_session` (per-test rollback) fixtures. Unit tests for pure logic use `Service.__new__(Service)` pattern (no DI). Integration tests use fixtures + `@pytest_asyncio.fixture`. Test paths: `tests/unit/`, `tests/integration/`, `tests/e2e/`.

**API routes:** Depends-based DI (`get_db`, `get_{repo}`), enum validation → `HTTPException(422)`, response builder helper functions (e.g., `_book_to_response`), eager loading via `selectinload()`. New routers registered in `backend/app/api/main.py` lines 171-190.

**Config:** `Settings` nested BaseSettings, `env_prefix="BOOKCOMPANION_"`, `env_nested_delimiter="__"`. `model_post_init()` for defaults + YAML merge. New settings file must hook here for XDG overlay.

**CLI DI:** `@asynccontextmanager get_services()` in `cli/deps.py` — lazy imports + soft-fail. Services registered as dict values.

**Frontend:** Pinia stores use `ref<Set<>>` for dedup + reconcile. TopBar already handles book-title for reader routes (already has 'book-summary' special-case). Existing primitives: `SkeletonLoader`, `EmptyState`, `LoadingSpinner`, `PresetPickerModal`. Deps: DOMPurify, markdown-it, Pinia, Vue Router 4, Tailwind v4 already installed. No `@vueuse/core`, no color-picker lib.

**CLI existing:** `tags_cmd.py` + `annotations_cmd.py` exist but NOT registered in `main.py`. `tags_cmd` accepts only "book" scope today (hardcoded).

**Frontend existing gaps:** No tag chip component, no cover fallback, no ContrastBadge, no useDebounceFn composable. All need to be built new.

| Data migration corrupts existing data (summaries, annotations, content_md) | Medium | Pre-backup before migration run (follows v1.4b pattern); narrow try/except catches only content-processing errors per NFR-06; each migration idempotent + has integration test on fixture DB |

| Custom-color WCAG calc off-by-one | Low | Unit tests for ContrastBadge against WCAG reference pairs |
| Tag name NOCASE uniqueness surprises with Unicode | Medium | Normalize on input (trim + whitespace collapse); SQLite NOCASE handles ASCII only — add explicit test for Unicode name (emoji / diacritic) showing it stores as-is |

  ```python
          out = dict(a)
  ```

  ```python
  def levenshtein(a: str, b: str) -> int:
      if len(a) < len(b): return levenshtein(b, a)
      if not b: return len(a)
      prev = list(range(len(b) + 1))
      for i, ca in enumerate(a):
          curr = [i + 1]
          for j, cb in enumerate(b):
              curr.append(min(prev[j+1]+1, curr[j]+1, prev[j] + (ca != cb)))
          prev = curr
      return prev[-1]

  def fuzzy_dedupe_against(candidates: list[str], against: list[str], threshold: int = 2) -> list[str]:
      against_lower = {s.lower() for s in against}
      out = []
      for c in candidates:
          cl = c.lower()
          if any(levenshtein(cl, al) <= threshold for al in against_lower):
              continue
          out.append(c)
      return out
  ```

  ```typescript
  // frontend/src/composables/useDebounceFn.ts
  export function useDebounceFn<T extends (...args: any[]) => void>(fn: T, ms: number) {
    let timer: ReturnType<typeof setTimeout> | null = null
    const debounced = (...args: Parameters<T>) => {
      if (timer) clearTimeout(timer)
      timer = setTimeout(() => { timer = null; fn(...args) }, ms)
    }
    debounced.cancel = () => { if (timer) { clearTimeout(timer); timer = null } }
    return debounced as T & { cancel: () => void }
  }
  ```

  ```typescript
    const h = hex.replace('#', '')
    const r = parseInt(h.slice(0,2), 16); const g = parseInt(h.slice(2,4), 16); const b = parseInt(h.slice(4,6), 16)
  ```
