### T1: Schema Migration M1 — BookSection failure cols + ProcessingJob partial UNIQUE + pid

**Goal:** Add the five new `book_sections` columns, the `processing_jobs.pid` column (if not already present), and the partial UNIQUE index on `(book_id) WHERE status IN ('PENDING','RUNNING')`.
**Spec refs:** §10.1, FR-A7.4, FR-A7.6

**Files:**
- Create: `backend/app/migrations/versions/<hash>_v1_4a_schema.py`
- Modify: `backend/app/db/models.py` (SQLAlchemy columns matching the migration)
- Create: `backend/tests/integration/test_migration_v1_4.py`

**Steps:**

- [ ] Step 1: Read current `ProcessingJob` model to confirm whether `pid` column exists already (prior commits added it but verify).
  Run: `grep -n "pid" backend/app/db/models.py | head`

- [ ] Step 2: Generate migration scaffold.
  Run: `cd backend && uv run alembic -c app/migrations/alembic.ini revision -m "v1 4a schema"`

- [ ] Step 3: Write `upgrade()` in the new migration:
  ```python
  def upgrade():
      with op.batch_alter_table("book_sections", schema=None) as batch:
          batch.add_column(sa.Column("last_failure_type", sa.String(64), nullable=True))
          batch.add_column(sa.Column("last_failure_message", sa.Text, nullable=True))
          batch.add_column(sa.Column("last_attempted_at", sa.DateTime(timezone=True), nullable=True))
          batch.add_column(sa.Column("attempt_count", sa.Integer, nullable=False, server_default="0"))
          batch.add_column(sa.Column("last_preset_used", sa.String(200), nullable=True))
          batch.create_index("ix_book_sections_last_attempted_at", ["last_attempted_at"])
      # Conditional pid: add only if not present
      if not _has_column("processing_jobs", "pid"):
          op.add_column("processing_jobs", sa.Column("pid", sa.Integer, nullable=True))
      op.create_index(
          "ix_processing_jobs_one_active_per_book",
          "processing_jobs",
          ["book_id"],
          unique=True,
          sqlite_where=sa.text("status IN ('PENDING','RUNNING')"),
      )
  ```

- [ ] Step 4: Write `downgrade()` inverting upgrade (drop columns, drop index). Use `batch_alter_table` for SQLite.

- [ ] Step 5: Reflect the new columns in `backend/app/db/models.py` `BookSection` (SQLAlchemy Mapped fields mirroring the migration).

- [ ] Step 6: Write failing tests in `backend/tests/integration/test_migration_v1_4.py`:
  ```python
  async def test_m1_applies_and_downgrades_cleanly(tmp_db):

      cols = await column_names(tmp_db, "book_sections")
      assert {"last_failure_type","last_failure_message","last_attempted_at","attempt_count","last_preset_used"} <= set(cols)
      # Partial UNIQUE: insert two RUNNING jobs for same book, expect IntegrityError on second

              await conn.execute(sa.text("INSERT INTO processing_jobs (book_id, status, step) VALUES (1, 'RUNNING', 'summarize')"))
      await run_alembic("downgrade", "-1")
      cols = await column_names(tmp_db, "book_sections")
      assert "last_failure_type" not in cols
  ```
