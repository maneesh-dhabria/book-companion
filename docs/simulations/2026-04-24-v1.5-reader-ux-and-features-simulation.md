## 1. Scope

**In scope:**
- Database: tags + taggables schema revival, `Annotation.prefix/suffix` columns, `Book.suggested_tags_json` + `last_summary_failure_*` columns, 6 Alembic migrations (3 schema + 3 data)
- API contracts: 13 endpoints (extended `GET /books/:id`, tag CRUD book+section, tag autocomplete + suggest, extended annotation create, settings GET/PATCH, config/models, suggested-tags PATCH, SSE event extension)
- Frontend: new `BookOverviewView` at `/books/:id`, `NoteCompositionPanel`, `CoverFallback`, `TagChip`/`TagChipInput`, `ContrastBadge`, `StickySaveBar`, highlight injector; extended `ReaderSettingsPopover` (6 presets + curated palettes + Custom…), extended `ContextSidebar` (defaultTab), extended `SettingsView` (LLM section)
- CLI: 5 new/extended command groups — `tag`, `model`, `annotations list`, `summarize --only-pending`, `doctor --orphan-tags`
- SSE: `processing_completed` extended with `suggested_tags[]` for book-summary jobs
- Config: new `backend/app/config/models.yaml` (startup-loaded, restart-to-reload)

**Out of scope (non-goals, not flagged as gaps):**
- Cross-device settings sync
- Multi-color highlights (single theme-aware yellow v1.5; `color` column reserved for v1.6)
- Section-level AI tag seeding (book-level only, one-shot)
- On-demand AI re-suggestion (one-shot at first successful book-summary)
- Library-wide tag taxonomy management UI (rename/merge/delete-unused)
- Reader-theme / highlight-rendering CLI commands (browser-only state)
- Annotation create/edit/delete via CLI (read-only `list` only)
- Per-summarization-job model override (Settings-only global default)

**Companion specs:** none (single-file spec).
**Anticipated downstream consumers:** none (single-user tool).

## 6. Pseudocode (2 targeted flows)

### Flow 1: Hybrid highlight anchor resolution (FR-C5.2)

```
Flow: Hybrid highlight anchor — match annotation to text range in rendered HTML
Entry: MarkdownRenderer post-sanitize, before final render to DOM

FUNCTION resolve_anchor(plaintext, annotation):
  # 1. Offset-direct match (happy path, ~99%)
  candidate = plaintext.slice(annotation.text_start, annotation.text_end)
  IF candidate == annotation.selected_text:
    RETURN (annotation.text_start, annotation.text_end)

  # 2. Fallback: enumerate all occurrences of selected_text
  indexes = []
  i = 0
  WHILE i < plaintext.length:
    j = plaintext.indexOf(annotation.selected_text, i)
    IF j == -1: BREAK
    indexes.push(j)
    i = j + 1

  IF indexes.length == 0:
    console.debug("anchor_miss", {id: annotation.id, reason: "no_occurrences"})
    RETURN null

  # 3. Disambiguate with prefix/suffix context
  FOR EACH idx IN indexes:
    end_idx = idx + annotation.selected_text.length
    prefix_ok = (
      annotation.prefix == "" OR annotation.prefix == null OR
      plaintext.slice(idx - annotation.prefix.length, idx) == annotation.prefix
    )
    suffix_ok = (
      annotation.suffix == "" OR annotation.suffix == null OR
      plaintext.slice(end_idx, end_idx + annotation.suffix.length) == annotation.suffix
    )
    IF prefix_ok AND suffix_ok:
      RETURN (idx, end_idx)   # First match wins

  # 4. No disambiguation possible
  console.debug("anchor_miss", {id: annotation.id, reason: "context_mismatch"})
  RETURN null
```

**DB calls:** none — operates on already-fetched Annotation rows + already-rendered HTML.

**State transitions:** none — pure function.

**Error branches:**
- No occurrences of `selected_text` → log + skip annotation (inline highlight not rendered; sidebar row still visible)
- Context mismatch → log + skip (same behavior)
- Runtime exception (should not happen for pure string ops) → propagate; `NFR-07` says never throw, so callers wrap in try/catch as defense in depth

**Concurrency notes:**
- Function is pure; safe to call in parallel over annotations
- Result is consumed by the injector which mutates the DOM — DOM mutation is single-threaded (Vue's render cycle), no shared-state concerns

### Flow 2: Idempotent tag add (FR-E2.5) + accept-as-cleanup (FR-E4.5)

```
Flow: Add a tag to a Book (or Section); dedupe with suggested_tags if accepting a suggestion
Entry: POST /api/v1/books/:id/tags  OR  POST /api/v1/sections/:id/tags

FUNCTION add_tag(scope, entity_id, raw_name, color):
  # 1. Normalize name (FR-E1.4)
  name = raw_name.strip()
  name = re.sub(r"\s+", " ", name)
  IF name == "":
    RETURN 400, {"detail": "name cannot be empty"}
  IF name.length > 200:
    RETURN 400, {"detail": "name too long (max 200)"}

  # 2. Verify entity exists
  entity = SELECT * FROM (books OR book_sections) WHERE id = entity_id
  IF entity is null:
    RETURN 404

  BEGIN TRANSACTION

    # 3. Get-or-create Tag (NOCASE unique)
    tag = SELECT * FROM tags WHERE name = :name COLLATE NOCASE
    IF tag is null:
      tag = INSERT INTO tags (name, color, created_at) VALUES (:name, :color, now())
      is_new_tag = TRUE
    ELSE:
      is_new_tag = FALSE

    # 4. Get-or-create Taggable (composite PK)
    existing_taggable = SELECT 1 FROM taggables
      WHERE tag_id = tag.id
        AND taggable_type = :scope
        AND taggable_id = entity_id
    IF existing_taggable:
      http_status = 200   # idempotent retry
    ELSE:
      INSERT INTO taggables (tag_id, taggable_type, taggable_id, created_at)
        VALUES (tag.id, :scope, entity_id, now())
      http_status = 201

    # 5. Accept-as-cleanup (FR-E4.5) — only for scope='book'
    IF scope == "book":
      book = SELECT suggested_tags_json FROM books WHERE id = entity_id
      IF book.suggested_tags_json contains (NOCASE) :name:
        new_list = book.suggested_tags_json.filter(n -> normalize(n).lower() != name.lower())
        UPDATE books SET suggested_tags_json = :new_list WHERE id = entity_id

  COMMIT

  RETURN http_status, tag_as_json(tag)
```

**DB calls:**
- SELECT from `books` or `book_sections` (entity existence check)
- SELECT from `tags` WHERE name NOCASE (get-or-create)
- INSERT INTO `tags` (only if new)
- SELECT from `taggables` WHERE composite PK (existence check)
- INSERT INTO `taggables` (only if new)
- SELECT `suggested_tags_json` FROM `books` (only for scope='book')
- UPDATE `books.suggested_tags_json` (only if accepted name appears)

**State transitions:**
- `Book.suggested_tags_json`: `[a, b, c]` → `[a, c]` when `b` is accepted as an applied tag

**Error branches:**
- Empty name after normalization → 400 "name cannot be empty" (no DB write)
- Name > 200 chars → 400 "name too long"
- Entity not found → 404 (no DB write)
- Concurrent duplicate INSERT into `tags` (race between `SELECT` and `INSERT`) → second INSERT raises `IntegrityError` on NOCASE UNIQUE → caller retries the SELECT + COMMIT path (RETRY-once pattern) → second attempt sees the tag exists, reuses it. Race window is microseconds; one retry suffices.
- Transaction rollback on any unexpected exception → no partial state committed

**Concurrency notes:**
- Transaction isolation: SQLite default (SERIALIZABLE under WAL with BEGIN IMMEDIATE). Single-writer model means the get-or-create race is bounded to the narrow SELECT→INSERT window within one transaction.
- Idempotency is enforced by: (a) NOCASE UNIQUE on `tags.name`, (b) composite PK on `taggables`. Both are DB-level guards so even if the service-layer check races, the DB rejects the second insert and the caller retries cleanly.
- `suggested_tags_json` mutation is inside the same transaction as the Taggable insert — cannot observe "tag applied but suggestion still shown" state.
