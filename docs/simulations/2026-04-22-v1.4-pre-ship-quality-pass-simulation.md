## 1. Scope

- **In scope:**
  - Backend API additions + modifications (`/summarize/presets`, `/books/:id/book-summary`, `/llm/status`, `/annotations`, `/books/:id/sections/:sid`, `/books/:id/summarize` scope=failed)
  - Backend services (SummarizerService per-section commits, stderr capture with timeout wrapper, PresetService exposure, orphan-job sweep)
  - SSE / event emission contract (additive fields on `section_failed` and `processing_completed`, applicability table per job type)
  - SQLite schema + migrations (two Alembic revisions, BackupService integration, partial UNIQUE index for in-flight jobs, `pid` column)
  - LLM subprocess integration (Claude + Codex CLI stderr capture, `cli_timeout`, `cli_not_found`)
  - Vue SPA (router deep-link fix, PresetPicker/PresetPickerModal/PresetGrid, BookSummaryView full implementation, MarkdownRenderer extraction, ChatMessage markdown, ChatScopeSelector, AnnotationsView with source context, CommandPalette loading/empty states, document.title, date formatting, casing)

- **Out of scope (deferred to separate spec / later phase):**
  - CLI surface — no changes to `bookcompanion <command>`
  - Authentication / multi-user / multi-tenant — personal single-user tool
  - External integrations (third-party APIs, webhooks) — none
  - Observability platform (metrics/counters/traces) — v1.5+ consideration

- **Companion specs:** None — this spec is all-in-one (backend + frontend).

- **Downstream consumers anticipated:** None external. Future v1.5 may add book-summary batch generation + per-preset analytics, but no forward-compat constraints required today.

## 4. Artifact Fitness Findings

### 4.1 Data & Storage

- **B-1** (minor, accepted) — Failure history kept only as `attempt_count + last_attempted_at` on `BookSection`. Prior-attempt details live only in the rotating log file. Accepted: full history in `summarization-failures.log`; DB only surfaces the current-state-of-affairs for UI.
- **B-2** (resolved by FR-C1.7) — Book-level failure now carried on `ProcessingJob.error_message`.
- **B-3** (resolved in part by FR-A7.6) — Partial UNIQUE index closes the TOCTOU window on in-flight jobs. No other table warranted versioning.
- **B-4** (patched) — `summarization-failures.log` format spec'd as plain text (`ISO | section_id | book_id | type | stderr_10k`).

### 4.2 Service Interfaces

- **B-5** (patched) — `default_id` rule: always `"practitioner_bullets"` in v1.4; first sorted preset if missing.
- **B-6** (patched) — 404 on `POST /book-summary` for unknown book_id.
- **B-7** (patched) — SSE event applicability table added to §9.2.1 (book-summary jobs emit `processing_*` only, not `section_*`).
- **B-8** (minor, accepted) — `/annotations` JOIN on book + section titles not optimized for 1000+ annotations. Acceptable at personal-tool scale; reconsider if list view becomes paginated.

### 4.3 Behavior / State Machines

- **B-9** (minor, accepted) — No `CANCELLED` state on `ProcessingJob`. User-initiated cancellation is a v1.5 feature.
- **B-10** (minor, accepted) — `BookSection`'s implicit 4-flag state product is workable; transitions explicit via FR-A5.0, FR-A4.1, FR-C4.3. No formal state-machine diagram needed.
- **B-11** (patched as FR-C4.3) — Failure columns cleared on successful retry.
- **B-12** (patched as FR-C1.8) — Post-success wiring from ProcessingJob → Book.default_summary_id.

### 4.4 Interface (UI)

- **B-13** (patched as FR-D1.5) — Loading spinner while loadBook awaits.
- **B-14** (patched, note in FR-B1.3) — Back-button behavior handled by existing param watcher.
- **B-15** (patched as FR-D1.4) — Optimistic rollback on chat-send failure.
- **B-16** (minor, accepted) — Keyboard nav (Tab/Enter/Escape) for PresetPickerModal + ChatScopeSelector. Covered in generic NFR-03 for now; concrete per-component spec deferred.

### 4.5 Wire-up

### 4.6 Operational

- **B-17** (minor, accepted) — No metrics / counters / structured observability. Full-stderr log is the only first-class observability artifact. v1.5+ may add Prometheus-style counters if needed; not today.

## 6. Pseudocode

### 6.1 Flow: `summarize_section()` (Flow 1)

Entry: `POST /books/:id/summarize` (scope=section or via book-level iteration)

```
FUNCTION summarize_section(section_id, preset_name, job_id, book_id):
  # FR-A5.0: SQL-atomic preset+counter update BEFORE subprocess
  db.execute("""
    UPDATE book_sections
    SET last_preset_used = :preset,
        attempt_count = attempt_count + 1,
        last_attempted_at = :now
    WHERE id = :id
  """, {preset, now, id: section_id})
  db.commit()

  event_bus.publish(job_id, "section_started", {section_id, title, index, total})

  preset = preset_service.load(preset_name)

  TRY:
    # FR-A4.5 subprocess timeout wrapper
    proc = await asyncio.create_subprocess_exec(...)
    stdout, stderr = await asyncio.wait_for(
      proc.communicate(), timeout=settings.llm.timeout_seconds
    )
    IF proc.returncode != 0:
      IF proc.returncode == -signal.SIGKILL:
        RAISE SubprocessKilled(stderr)
      RAISE SubprocessNonZeroExit(proc.returncode, stderr)

    result = parse_json_schema(stdout)  # may raise SchemaParseError
    summary_md = result.summary_md

    # FR-A6.1 empty-output enforcement at service layer
    IF summary_md.strip() == "":
      RAISE EmptySummaryError()

    # Per-section commit (FR-A7.1)
    summary = Summary(
      content_type='section', content_id=section_id,
      preset_name=preset_name, summary_md=summary_md, ...
    )
    db.add(summary); db.flush()  # get summary.id

    # FR-C4.2 auto-default on first non-empty
    IF section.default_summary_id IS NULL:
      section.default_summary_id = summary.id

    # FR-C4.3 clear failure state on success (B-11/P-2 resolution)
    section.last_failure_type = NULL
    section.last_failure_message = NULL

    db.commit()

    event_bus.publish(job_id, "section_completed", {section_id, title, index, total, elapsed_seconds})

  EXCEPT SubprocessNonZeroExit as e:
    handle_failure(section_id, job_id, "cli_nonzero_exit", e.stderr)
  EXCEPT asyncio.TimeoutError:
    proc.kill()
    handle_failure(section_id, job_id, "cli_timeout", f"Timed out after {settings.llm.timeout_seconds}s")
  EXCEPT FileNotFoundError:  # claude / codex missing from PATH
    handle_failure(section_id, job_id, "cli_not_found", "CLI binary not found on PATH")
  EXCEPT SchemaParseError as e:
    handle_failure(section_id, job_id, "schema_parse_failed", e.message, e.raw)
  EXCEPT EmptySummaryError:
    handle_failure(section_id, job_id, "empty_output", "LLM returned empty summary")
  EXCEPT Exception as e:
    handle_failure(section_id, job_id, "db_error", str(e)[:500])
    RAISE  # bubble up — unexpected


FUNCTION handle_failure(section_id, job_id, type_code, message, full_stderr=""):
  truncated = message[:500]
  db.execute("""
    UPDATE book_sections
    SET last_failure_type = :t, last_failure_message = :m
    WHERE id = :id
  """, {t: type_code, m: truncated, id: section_id})
  db.commit()

  IF settings.llm.stderr_log_enabled:
    failure_logger.error(
      f"{utcnow_iso()} | section_id={section_id} | book_id={book_id} | "
      f"type={type_code} | {full_stderr[:10000]}"
    )

  event_bus.publish(job_id, "section_failed", {
    section_id, title, index, total,
    error: truncated,
    error_type: type_code,
    error_message_truncated: truncated
  })
```

**DB calls:** SELECT BookSection; UPDATE BookSection pre-subprocess (atomic counter+preset); INSERT Summary on success; UPDATE BookSection default+clear-failure on success; UPDATE BookSection failure cols on failure.

**State transitions:**
- BookSection on success: has_summary → true, default_summary_id set, last_failure_* → NULL
- BookSection on failure: last_failure_type set to typed code, attempt_count already incremented

**Error branches:** six typed paths (`cli_nonzero_exit`, `cli_timeout`, `cli_not_found`, `schema_parse_failed`, `empty_output`, `db_error`), each maps to a failure record + SSE emission.

**Concurrency notes:** per-section commit (FR-A7.1) keeps write txn short → reading-state PUT succeeds (G7). Two concurrent jobs on different books: SQLite WAL + busy_timeout=5s + per-section commits → serial at writer but tolerable. `attempt_count = attempt_count + 1` is SQL-side atomic (P-1 resolution).

### 6.2 Flow: Migration 2 (data backfill + empty cleanup)

Entry: Alembic upgrade for revision `v1_4_default_summary_backfill_and_empty_cleanup`

```
FUNCTION upgrade():
  logger.info("migration_2_started")

  # FR-A6.3 pre-migration backup
  TRY:
    backup_path = data_dir / f"library.db.pre-v1-4-{timestamp()}.bak"
    backup_service.create_backup(dest=backup_path)
    logger.info(f"migration_2_backup_ok path={backup_path}")
  EXCEPT Exception as e:
    logger.error(f"migration_2_backup_failed error={e}")
    RAISE  # aborts; Alembic version stays at M1; FR-A6.4 self-heal on next boot

  # FR-A6.4 single-transaction data work
  conn = op.get_bind()
  counts = {"sections_backfilled": 0, "empty_deleted": 0, "sections_marked_failed": 0}

  # Part A: empty-summary cleanup
  empty_rows = conn.execute(sa.text("""
    SELECT id, content_id AS section_id, book_id, created_at
    FROM summaries
    WHERE content_type = 'section'
      AND (summary_md IS NULL OR TRIM(summary_md) = '')
  """)).fetchall()

  FOR row IN empty_rows:
    # Null default_summary_id if pointed to this empty row
    conn.execute(sa.text("""
      UPDATE book_sections SET default_summary_id = NULL
      WHERE id = :sid AND default_summary_id = :summary_id
    """), {sid: row.section_id, summary_id: row.id})

    # P-3 guard: don't overwrite newer non-empty-output failures
    conn.execute(sa.text("""
      UPDATE book_sections
      SET last_failure_type = 'empty_output',
          last_failure_message = 'Cleaned up empty summary from prior run',
          last_attempted_at = :created_at,
          attempt_count = COALESCE(attempt_count, 0) + 1
      WHERE id = :sid
        AND (last_failure_type IS NULL OR last_failure_type = 'empty_output')
    """), {sid: row.section_id, created_at: row.created_at})

    conn.execute(sa.text("DELETE FROM summaries WHERE id = :id"), {id: row.id})
    counts["empty_deleted"] += 1
    counts["sections_marked_failed"] += 1

  # Part B: auto-select default_summary
  sections_needing_default = conn.execute(sa.text("""
    SELECT bs.id AS section_id FROM book_sections bs
    WHERE bs.default_summary_id IS NULL
      AND EXISTS (
        SELECT 1 FROM summaries s
        WHERE s.content_type = 'section' AND s.content_id = bs.id
          AND TRIM(COALESCE(s.summary_md, '')) != ''
      )
  """)).fetchall()

  FOR row IN sections_needing_default:
    latest = conn.execute(sa.text("""
      SELECT id FROM summaries
      WHERE content_type = 'section' AND content_id = :sid
        AND TRIM(COALESCE(summary_md, '')) != ''
      ORDER BY created_at DESC LIMIT 1
    """), {sid: row.section_id}).first()
    IF latest:
      conn.execute(sa.text("""
        UPDATE book_sections SET default_summary_id = :sid WHERE id = :section_id
      """), {sid: latest.id, section_id: row.section_id})
      counts["sections_backfilled"] += 1

  logger.info(f"migration_2_done {counts}")


FUNCTION downgrade():
  # Data migration not reversible — restore from backup
  logger.warning("migration_2_downgrade_no_op — restore from library.db.pre-v1-4-*.bak")
```

**DB calls:** SELECT empty summaries; per-row: UPDATE book_sections (default_summary_id NULL); UPDATE book_sections (failure cols with P-3 guard); DELETE summaries; SELECT sections needing default; per-section: SELECT latest non-empty summary; UPDATE book_sections (default_summary_id set).

**State transitions:**
- Summary rows: empty → deleted
- BookSection.default_summary_id: NULL (pointing to deleted) → NULL → <new>
- BookSection.last_failure_type: NULL → 'empty_output' (unless newer failure present)

**Error branches:** Backup fails → abort, Alembic stays at M1. DB operation fails mid-transaction → rollback all Part A+B work, Alembic stays at M1. Next boot retries automatically (FR-A6.4).

**Concurrency notes:** Runs at startup before app accepts requests. External `sqlite3` session could block BEGIN after busy_timeout=5s → rollback → retry on next boot.

### 6.3 Flow: Retry failed sections batch (scope=failed)

Entry: `POST /books/:id/summarize {scope: "failed", preset_name: <user-picked>}`

```
FUNCTION post_summarize(book_id, body):
  preset = preset_service.load(body.preset_name)  # raises PresetError → 400

  # FR-A7.3 explicit in-flight check (belt)
  in_flight = db.scalar(
    sa.select(ProcessingJob).where(
      ProcessingJob.book_id == book_id,
      ProcessingJob.status.in_(['PENDING', 'RUNNING'])
    )
  )
  IF in_flight:
    RAISE HTTPException(409, "A summarization job is already running for this book")

  IF body.scope == "failed":
    failed = db.scalars(sa.select(BookSection).where(
      BookSection.book_id == book_id,
      BookSection.last_failure_type.isnot(None),
      BookSection.default_summary_id.is_(None)
    )).all()
    IF not failed:
      RAISE HTTPException(400, "No failed sections to retry")
    section_ids = [s.id for s in failed]
  ELIF body.scope == "section":
    section_ids = [body.section_id]
  ELIF body.scope == "book":
    section_ids = None  # summarizer walks all summarizable

  # FR-A7.6 DB constraint (suspenders) — partial UNIQUE on (book_id) WHERE status IN ('PENDING','RUNNING')
  TRY:
    job = ProcessingJob(book_id=book_id, status='PENDING', pid=os.getpid())
    db.add(job); db.commit()
  EXCEPT IntegrityError:
    # Race lost — another request just inserted a RUNNING job
    RAISE HTTPException(409, "A summarization job is already running for this book")

  asyncio.create_task(run_summarize_job(
    job.id, book_id, body.preset_name, section_ids=section_ids
  ))

  RETURN {"job_id": job.id}


FUNCTION run_summarize_job(job_id, book_id, preset_name, section_ids=None):
  job = db.get(ProcessingJob, job_id)
  job.status = 'RUNNING'; job.started_at = utcnow()
  db.commit()

  event_bus.publish(job_id, "processing_started", {book_id, job_id, scope})

  IF section_ids IS NULL:
    sections = db.scalars(sa.select(BookSection).where(
      BookSection.book_id == book_id,
      BookSection.is_summarizable  # derived from section_type
    ).order_by(BookSection.order_index)).all()
  ELSE:
    sections = db.scalars(sa.select(BookSection).where(
      BookSection.id.in_(section_ids)
    ).order_by(BookSection.order_index)).all()

  counts = {"completed": 0, "failed": 0, "skipped": 0}
  FOR i, section IN enumerate(sections):
    # scope=book idempotency
    IF section.default_summary_id IS NOT NULL AND has_non_empty_summary(section):
      event_bus.publish(job_id, "section_skipped", {section_id, title, index: i, total: len(sections), reason: "already_summarized"})
      counts["skipped"] += 1
      CONTINUE

    TRY:
      await summarize_section(section.id, preset_name, job_id, book_id)  # Flow 1
      counts["completed"] += 1
    EXCEPT Exception:
      counts["failed"] += 1
      # section_failed SSE already emitted by Flow 1's failure handler

  IF counts["completed"] == 0 AND counts["failed"] > 0:
    job.status = 'FAILED'
  ELSE:
    job.status = 'COMPLETED'
  job.completed_at = utcnow()
  db.commit()

  event_bus.publish(job_id, "processing_completed", {book_id, **counts, book_summary_id: None})
```

**DB calls:** SELECT in-flight ProcessingJob; INSERT ProcessingJob (may raise IntegrityError); SELECT BookSections; per-section: see Flow 1; UPDATE ProcessingJob status+completed_at.

**State transitions:** ProcessingJob PENDING → RUNNING → {COMPLETED | FAILED}. Sections: see Flow 1.

**Error branches:** 409 if in-flight (FR-A7.3) or IntegrityError (FR-A7.6). 400 for bad preset or no failed sections. Per-section failures captured by Flow 1; job continues. Orphan job on Python crash → swept at next server boot (FR-A7.4).

**Concurrency notes:** FR-A7.3 + FR-A7.6 close the TOCTOU race (P-5 resolution). Two tabs both POST: one 201, one 409. Two books: each has its own partial-index row — both proceed.

## 8. Accepted Risks

For each, the rationale captured at gap-disposition time:

- **S-3** — Stale deep-link on book re-import may show stale section content. *Rationale:* low-incidence — re-import is rare; user can hard-reload; E8 already documents it as a known limitation.
- **S-7** — Two-book concurrent summarize not explicitly performance-tested. *Rationale:* SQLite WAL + per-section commits + busy_timeout=5s is sound; failure mode at worst is a transient 5xx which FR-A7.1 + NFR-02 already require to be zero. Personal scale.
- **S-11** — Cross-tab coordination for same-book job state (Tab B doesn't know Tab A started a job). *Rationale:* 409 handling on the losing click is sufficient UX; no real-time cross-tab sync infrastructure in v1.4; personal tool = rare to have two tabs.
- **S-12** — No performance NFR for markdown render on large `content_md` (500KB+). *Rationale:* rare in non-fiction practice (typical chapter 10-50KB); markdown-it is fast; unmeasured but unlikely problem.
- **S-17** — FR-F4.1 regex false-positive on short acronym section titles (e.g., "MCP" in a very short chapter). *Rationale:* 200-char content guard catches most cases; false positive misclassifies a chapter as `title_page` (no summary generated); user can manually re-trigger; low blast radius.
- **B-1** — Failure history limited to latest timestamp in DB. *Rationale:* full history lives in `summarization-failures.log` rotating file (B-4); DB carries UI-facing state only; more elaborate history table is scope creep for v1.4.
- **B-3** — No optimistic-lock `version` column anywhere. *Rationale:* partial UNIQUE index on ProcessingJob (FR-A7.6) handles the one concurrency scenario that matters. Other writes are single-owner (user) or single-writer (summarizer).
- **B-8** — `/annotations` JOIN on book + section titles may N+1 at scale. *Rationale:* personal-tool scale. If the annotations list exceeds ~1000 rows with perf concerns, revisit in v1.5 with pagination.
- **B-9** — No CANCELLED state on ProcessingJob. *Rationale:* user-initiated cancellation is a distinct feature; requires mid-flight subprocess kill + DB consistency work. v1.5+.
- **B-10** — BookSection's implicit 4-flag state product instead of formal state machine. *Rationale:* v1.4 scope keeps `is_summarizable` / `has_summary` / `has_failure` / `has_preset_used` as orthogonal flags; FRs A5.0, A4.1, C4.3 cover all transitions; a diagram would add documentation mass without new clarity.
- **B-16** — Keyboard nav for `PresetPickerModal` + `ChatScopeSelector` not component-spec'd. *Rationale:* NFR-03 generic a11y requirement covers it; concrete per-component keyboard spec can live in design review during implementation.
- **B-17** — No metrics / counters / structured observability platform. *Rationale:* full-stderr log covers 95% of debug needs; metrics infra is a platform decision for v1.5 / post-personal-scale.
- **W-2** — `/annotations` response shape change not explicitly declared backward-compatible. *Rationale:* no external consumers; frontend upgrades atomically with backend; declaration is redundant.
- **W-3** — Annotation row click navigation depends on FR-B1.1 deep-link fix. *Rationale:* acceptable implicit coupling — if B1.1 regresses, the test suite catches it upstream of W-3.
- **P-4** — `library.db.pre-v1-4-*.bak` files accumulate on retry loops. *Rationale:* at worst a handful of ~6MB files; user can manually prune; rare on disk-full scenarios.
