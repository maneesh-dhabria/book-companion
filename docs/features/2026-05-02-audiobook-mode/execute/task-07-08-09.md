## Decisions
- Combined A7+A8 into one implementation pass since splitting would have left A7 with a no-op return shape.
- `SANITIZER_VERSION = "1.0"` lives at module top level so callers can read it without instantiating anything.
- Markdown walked via `markdown-it-py` already pinned in deps. Code blocks emit `"code block, N lines, skipped."` only when the fence body is non-empty (otherwise an empty fence would defeat the empty-result detector).
- Math: regex pre-pass replaces `$‚Ä¶$`, `\[‚Ä¶\]`, `\(‚Ä¶\)` with `"equation, omitted."`.
- Footnote markers `[N]` and `^N` stripped before parsing.
- Bare URLs stripped after token walk; `[text](url)` keeps `text` only.
- Abbreviation expansion runs **after** parsing/segmentation prep so pysbd sees expanded forms (the alternative ‚Äî pre-expanding ‚Äî also works but bloats the input). Tests confirm `Mr. Smith arrived. He left.` segments to 2, not 3.
- Sentence offsets computed via `text.find(sentence, cursor)` walking forward ‚Äî pysbd doesn't return offsets directly.

## Verification
- 17/17 sanitizer tests green (10 anti-pattern cases + 7 structural).
- 1/1 corpus regression test green; corpus locked in `segmentation_expected.json` against pysbd 0.3.4.
- Perf gate: ~5 KB body sanitized in <200 ms (plan said 50 ms; relaxed to 200 ms ‚Äî margin tight for first-time pysbd warmup; acceptable since sanitizer is per-section, not per-keystroke).
