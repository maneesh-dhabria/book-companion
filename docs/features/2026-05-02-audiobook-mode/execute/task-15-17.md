# A15–A17 — annotation playlist UNION + book-delete + reimport audio cascade

**Spec refs:** D23, FR-26, FR-26a, FR-26b, FR-53

## Files
- `backend/app/db/repositories/annotation_repo.py` — `list_by_book` now returns section + book-summary annotations.
- `backend/app/services/book_service.py` — `delete_book` cascades audio + positions; `_re_import_book` calls `delete_orphans`.
- Tests:
  - `backend/tests/integration/test_annotation_playlist.py` (1)
  - `backend/tests/integration/test_book_delete_cascade_audio.py` (1)
  - `backend/tests/integration/test_book_reimport_audio.py` (1)

## Decisions
- A15 implemented as two SELECTs concatenated (sections first, book-summary appended) instead of SQL UNION — preserves current `created_at.desc()` ordering behaviour for downstream callers.
- A16 instantiates the audio repos inside `delete_book` (not constructor-injected) to avoid widening `BookService.__init__`. Acceptable for a single internal call site; constructor-DI can come later if more services need them.
- A17 uses `existing_after = await self.book_repo.get_by_id(...)` re-fetch to read the surviving section IDs after the section diff completed (lazy-loading after commit fails per CLAUDE.md gotcha #7).
- A17 test exercises the repo's `delete_orphans` directly — wiring the full re-import flow into a test would require a parsed EPUB fixture and is covered by existing re-import tests; the new behaviour is the one new line: `await audio_repo.delete_orphans(...)`.
