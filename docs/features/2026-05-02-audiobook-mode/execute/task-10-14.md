## Decisions
- Registered `kokoro` pytest marker; default `addopts = "-m 'not slow and not kokoro'"` so heavy Kokoro tests are opt-in (per plan P7).
- `KokoroProvider._ensure_model_downloaded` is a `@staticmethod` so it's monkey-patchable from the factory tests.
- Provider takes `model_dir` constructor arg; factory wires it from `Settings.data.directory / models / tts`.
- `synthesize(text, voice)` calls our sanitizer and segments first; `synthesize_segmented(sentences, voice)` is the lower-level entry point (the AudioGenService in B2 will call segmented form).
- ID3 tagger handles both fresh files and re-tag overwrites by `delall(...)` of the 5 frames before re-add.
- `huggingface_hub` is imported inside `_ensure_model_downloaded` (not at module top) so the package is optional — only required when Kokoro is in use.
- Did **not** run `@pytest.mark.kokoro` tests this session (they require an 80 MB model download + ffmpeg). Code paths exercised by mocked download + factory tests + sanitizer tests; live model tests deferred to a manual `pytest -m kokoro` run.
