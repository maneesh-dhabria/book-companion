Search — Design Summary

  • ⌘K Command Palette: Centered modal, instant results grouped by type (Books, Sections, Concepts, Annotations). Keyboard navigation (↑↓ navigate, ↵ open, ⇧↵ full results). Match highlighting in bold. Empty state shows recent searches + quick actions.
  • Full Results Page: Left sidebar filters (Source Type with counts, Book filter, Tag filter). Results grouped by book with match count. Each result shows: type badge (color-coded), title, snippet with highlights, relevance score. Filter sidebar auto-collapses when result count < 20, showing a "Show filters" toggle link instead; auto-expands when results > 20.
  • Hybrid search: BM25 (keyword) + semantic (embedding) + RRF fusion. Indicated in footer/header so user knows it's more than keyword matching.
  • Result types: section (blue), summary (green), concept (purple), annotation (orange) — each color-coded consistently.
  • Result actions: Click opens the source directly (section reader, concept tooltip, annotation in context). "Show N more" expands within book group.
  • Sort: Relevance (default), Date, Book order.
