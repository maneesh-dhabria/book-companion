Behavior summary. When the user lands on Step 3, the frontend issues a server-side preflight that resolves the selected provider's binary on the host's executable PATH. If the binary is missing, the error banner appears, the preset cards and Start button are disabled, and clicking Fix in Settings → LLM deep-links to the Settings page (see wireframe 07). No job is queued. Note that since wireframe 07's revised Settings page only lists detected binaries in the Provider dropdown, this preflight failure now only happens if the binary was uninstalled after the provider was selected — a much rarer edge case than the original alias-typed-into-free-form-field bug.
